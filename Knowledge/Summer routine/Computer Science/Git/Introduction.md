## **Git – Short Introduction**

## What is Git?

**Git** is a **distributed version control system (VCS)** used to track changes in code and collaborate with others.  
It allows you to save snapshots of your project, revert to previous versions, and work in parallel without conflicts.

Created by **Linus Torvalds** in 2005.

---

## Core Concepts

### Repository (Repo)

A Git-tracked project folder.

- **Local repository** – on your computer
    
- **Remote repository** – hosted online (e.g., **GitHub**, **GitLab**)
    

---

### Commit

A **snapshot** of your project at a specific time.  
Each commit has:

- Unique ID (hash)
    
- Author
    
- Message
    
- Timestamp
    

---

### Branch

A separate line of development.

- `main` (or `master`) = default branch
    
- Feature branches = new functionality without affecting main code
    

---

### Merge

Combining changes from one branch into another.

---

### Clone

Copying a remote repository to your local machine.

---

### Pull

Fetching and merging changes from a remote repository.

---

### Push

Uploading your local commits to a remote repository.

---

### Staging Area (Index)

A preparation area where you select changes before committing.

Workflow:

```Git
Working Directory → Staging Area → Commit → Remote
```
